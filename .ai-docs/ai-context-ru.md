# Контекст и Инструкции для AI Агента (Cinora)

Приветствую, коллега AI-Агент! 🤖
Если ты читаешь этот файл, значит тебя запустили на новой машине для продолжения работы над проектом **Cinora**. В этом документе собраны ключевые правила и контекст, которые помогут тебе бесшовно продолжить разработку.

## 🎯 Текущая задача
В данный момент мы находимся на **Фазе 2** плана разработки (см. `roadmap.md`).
Ближайшие задачи — создание социальных функций: Системы комментариев и 5-звездочного рейтинга (вместо лайков).

## 🏗 Архитектура и Стек Технологий
- **Фреймворк**: Next.js 14 (App Router)
- **Язык**: TypeScript
- **База Данных**: Prisma ORM с SQLite (для локальной разработки).
  - *Важно*: Локальная БД SQLite (`dev.db`) НЕ отслеживается в Git (добавлена в `.gitignore`). Пользователь должен сам скопировать актуальный `dev.db` с сервера, либо тебе нужно подсказать ему запустить `npx prisma db push`, чтобы создать пустую базу локально.
- **Авторизация**: NextAuth.js (Credentials Provider / вход по email).
- **Стилизация**: Tailwind CSS (Светлый текст на темном фоне, премиальный дизайн).

## 📂 Структура папок
- `/src/app/(auth)` - Страницы входа и регистрации.
- `/src/app/api` - Бэкенд API-маршруты (NextAuth, Cron-скрипты, Админка, Рейтинги).
- `/src/components` - React-компоненты. (Не забывай добавлять `"use client"` в начале файла, если там используются хуки или обработчики событий).
- `/prisma/schema.prisma` - Главный файл со схемой базы данных.

## 💾 Устройство Базы Данных (Prisma)
У нас есть следующие основные модели:
- `User`: Данные пользователей и роли (`ADMIN` / `USER`).
- `Movie` / `Series`: Метаданные контента, которые мы забираем из TMDB/Кинопоиска.
- `VideoSource`: Внешние ссылки на плееры (CDN). Важно: в запросах на получение фильмов мы фильтруем контент без плееров через `where: { videoSources: { some: {} } }`.
- `Rating`: Лайки (а скоро и оценки звездами) от пользователей.
- `WatchHistory`: История просмотров, используется на главной странице для умной подборки "Герой-карусель".
- `Favorite`: Избранное (список "Буду смотреть").

## 📜 Главные Правила Разработки
1. **Используй конкретные инструменты**: Если тебе нужно прочитать файл, используй `view_file`. Если поменять код — `replace_file_content`. Избегай общих Bash-команд вроде `cat` или `sed`.
2. **Эстетика и дизайн КРИТИЧЕСКИ ВАЖНЫ**: Создавая новые UI-компоненты, всегда используй современный, премиальный дизайн. Применяй эффект стекла (glassmorphism — `bg-[#1a1a2e]/80 backdrop-blur-md`), плавные анимации и красивую типографику. Скучные и базовые интерфейсы неприемлемы!
3. **Осторожнее с базой данных**: Не запускай `prisma migrate dev`, так как локальный `dev.db` может быть точной копией продакшена. При изменении схемы используй `prisma db push`, но будь осторожен, чтобы не удалить существующие данные.
4. **Client vs Server Components**: По умолчанию все страницы в `/src/app` серверные. Если нужна интерактивность (кнопка лайка, форма отправки комментария), создавай отдельный клиентский компонент в `/src/components` и импортируй его на страницу.
5. **Переменные окружения (.env)**: Файл `.env` игнорируется в Git. Если у тебя проблемы с логином локально, убедись, что у пользователя в файле `.env` прописаны правильные пути (`NEXTAUTH_URL=http://localhost:3000` и `DATABASE_URL="file:./dev.db"`).
